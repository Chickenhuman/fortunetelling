# 사주풀이/운세 서비스 개발 계획서

- 프로젝트: `fortunetelling`
- 작성일: `2026-03-05`
- 마지막 업데이트: `2026-03-05`
- 개발 원칙: 기존 UI 유지, 모바일 우선, 단계별 출시

## 1) 목표

1. 모바일 인터페이스를 기본으로 하는 웹 서비스 우선 출시
2. 이후 앱 서비스로 확장 가능한 구조 확보
3. AI 분석(모델 교체/선택 가능), 회원 기능, 결제, 광고, 히스토리까지 순차 도입

## 2) 현재 상태 (As-Is)

1. 단일 페이지 UI(`index.html`) + `/api/analyze` 백엔드 연동 완료
2. OpenAI/Mock 제공자 분리 구조 존재 (`src/providers`)
3. 사주 계산은 `@fullstackfamily/manseryeok` 기반 엔진으로 교체 완료
4. 회원가입/로그인 API + 히스토리 저장/조회 API는 구현 완료
5. 결제 API + 결제 UI(요금제 선택/충전/지갑 조회)는 구현 완료
6. 광고 정책 API + 기본 광고 UI(무료/유료 분기)는 구현 완료
7. 다국어/모델선택 UI는 미구현

## 3) 아키텍처 방향 (To-Be)

1. 프론트: 모바일 우선 UI 유지 + 반응형 확장 (웹은 후순위)
2. 백엔드: 인증/결제/히스토리/광고/AI 분석을 API 모듈로 분리
3. AI: 공급자 추상화 유지, 모델 선택 기능(런타임 선택 + 운영 정책) 추가
4. 데이터: 베타는 로컬 서버 저장 중심, 정식은 계정 기반 서버 저장으로 전환
5. 앱 확장: 웹 API 재사용 가능한 구조로 앱 클라이언트 연결
